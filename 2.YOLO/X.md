这是一个**极具前瞻性**的选择！X-AnyLabeling 是目前标注界的新宠。

相比于“纯手工”的 LabelImg，X-AnyLabeling 最大的优势在于**AI 辅助标注（AI-Assisted Labeling）**。
简单说：**它能调用已经训练好的模型（比如 YOLOv8 或 SAM）帮你先自动标一遍，你只需要微调。** 这能把你们战队标注数据的效率提升 5-10 倍。

以下是保姆级的下载与使用教程。

---

### 第一部分：下载与安装

X-AnyLabeling 不需要复杂的环境配置，它直接提供了一个打包好的 `.exe` 文件，开箱即用（特别适合 Windows 用户）。

**1. 下载地址（GitHub Releases）：**

* **官方地址：** [https://github.com/CVHub520/X-AnyLabeling/releases](https://github.com/CVHub520/X-AnyLabeling/releases)
* **选择文件：**
找到最新的版本（比如 v2.x.x），在 "Assets" 里找到以 `.exe` 结尾的文件。
* 通常叫：`X-AnyLabeling-GPU-v...exe` (如果你有 N 卡)
* 或者：`X-AnyLabeling-CPU-v...exe` (如果你只用 CPU)
* *鉴于你之前的环境，如果网速允许，建议下载 CPU 版先跑起来，文件比较小。*



> **⚠️ 注意：** GitHub 在国内下载可能很慢。如果下载不动，可以使用“Gitee 镜像”搜索同名项目，或者使用 Steam++ 等工具加速 GitHub。

**2. 安装：**
它是一个**绿色软件**，不需要安装。下载下来是一个压缩包或直接是 exe，双击打开即可运行。

---

### 第二部分：界面与基础设置

打开软件后，你会看到一个比 LabelImg 现代得多的界面。

**1. 准备工作：**

* 点击左上角的 **"Open Dir"**（打开目录），选择你存放图片的文件夹（比如之前的 `dataset`）。
* 点击左上角的 **"Change Output Dir"**（更改输出目录），建议选同一个文件夹，或者新建一个 `labels` 文件夹。

**2. 开启自动保存：**

* 点击菜单栏 **File** -> 勾选 **Auto Save**。这样你切图的时候就不用每次按 Ctrl+S 了。

---

### 第三部分：核心功能——AI 自动标注（杀手锏）

这是你选择它的原因。我们让 AI 帮你干苦力。

**1. 加载模型：**

* 看左侧工具栏，找到一个 **“大脑”形状的图标** (AI Mode)。
* 点击它，会弹出一个列表。
* 选择 **Model** -> **YOLOv8** -> **YOLOv8n** (最轻量，下载最快)。
* *第一次选会提示下载模型，点击 OK 等待下载完成。*



**2. 一键标注：**

* 模型加载完后，点击“大脑”图标下方的 **“运行”按钮**（或者快捷键 `Ctrl + J`）。
* **奇迹发生：** 软件会自动识别图中的物体（比如人、车等），并自动画好框！

**3. 你的工作：**

* **如果框对了：** 只需要把类别名字改一下（比如它识别成 `person`，你双击改成 `robot`）。
* **如果没框住：** 你再手动补画一下。

---

### 第四部分：手动标注与工具

如果是 AI 识别不出来的特殊物体（比如你们比赛的装甲板），你需要手动标。

**1. 画矩形框 (检测任务)：**

* 快捷键：**`R`** (Rectangle)。
* 按住鼠标左键拖拽画框。
* 输入类别名称（如 `armor`）。

**2. 画多边形 (分割任务 - 进阶)：**

* 如果你以后做图像分割（Image Segmentation），需要沿着物体轮廓描点。
* 快捷键：**`P`** (Polygon)。

**3. 旋转框 (OBB - 机器人竞赛常用)：**

* 有些比赛（如 RoboMaster）需要识别旋转的物体（俯视图）。
* X-AnyLabeling 支持旋转框，这在 LabelImg 里是做不到的。
* 工具栏里选择 **"Rotated Box"**。

---

### 第五步：导出为 YOLO 格式 (关键！)

X-AnyLabeling 默认保存的是 JSON 格式（包含很详细的信息），但 YOLO 训练需要的是 `.txt` 格式。

**你需要做一次“导出”操作：**

1. 标完所有图片后。
2. 点击菜单栏左上角的 **"Export"** (导出图标，通常在保存旁边)。
3. 选择导出格式：**"Export to YOLO"**。
4. 在弹出的设置里：
* **Class Name File:** 它可以自动生成，或者你导入之前的 `classes.txt`。
* **Output Dir:** 选择导出位置。


5. 点击 OK。

它会生成 `train/images`, `train/labels` 这样的标准结构，你可以直接把这个文件夹喂给 YOLOv8 训练！

---

### 总结：你的新工作流

1. **拍照/收集数据** -> 放入文件夹。
2. 打开 **X-AnyLabeling**。
3. **(可选)** 加载 **YOLOv8n** 模型 -> `Ctrl + J` 自动预标注 -> 修正错误。
4. **(必须)** 手动补标漏掉的物体。
5. **Export to YOLO**。
6. 回到 VS Code 运行 `model.train()`。

这取决于你刚才**标注时画的是什么框**，以及你打算训练**什么类型的模型**。

对于你目前的任务（识别鼠标、水杯，或者是大多数机器人竞赛中的正向视角检测），**请选择第一个：**

### 👉 **`导出YOLO-Hbb标注`** (Horizontal Bounding Box)

#### 为什么选这个？

* **Hbb (水平边界框)**：这是最标准的目标检测格式。它生成的标签包含 `class, x_center, y_center, width, height`。
* **对应模型**：它对应的是标准的 `yolov8n.pt`（你刚才下载的那个）。
* **使用场景**：只要你刚才用的是**矩形工具 (快捷键 R)** 画的框，就必须选这个。

---

### 其他选项是干嘛的？（作为视觉组长你需要知道）

虽然这次不用，但为了应对未来的**机器人竞赛**，请记下这两个：

1. **`导出YOLO-Obb标注` (Oriented Bounding Box - 旋转框)**
* **场景**：如果你在做**RoboMaster**或者**自动驾驶**的俯视图，需要检测**倾斜的长条物体**（比如旋转的装甲板、放在地上的条形障碍物），并且你使用了**旋转框工具**进行标注。
* **对应模型**：需要使用专门的 `yolov8n-obb.pt` 模型来训练。
* **注意**：如果你刚才画的是普通矩形，选这个会报错或效果极差。


2. **`导出YOLO-Seg标注` (Segmentation - 实例分割)**
* **场景**：如果你不仅要框出物体，还要**沿着轮廓把物体“扣”出来**（像素级识别），并且你使用了**多边形工具 (快捷键 P)** 描点。
* **对应模型**：需要使用 `yolov8n-seg.pt`。

这个报错的意思非常简单直接：**你的软件现在的“户口本”（类别列表）里，查不到 'mouse' 这个名字。**

这是因为 YOLO 最终需要的不是“mouse”这个单词，而是数字（比如 `0`）。软件试图把你的标签转换成数字时，发现它的对照表里没有这个词，所以罢工了。

请按照以下步骤解决，**1分钟就能搞定**：

### 第一步：手动制作“户口本”

1. 在你的电脑桌面（或者项目文件夹里），**新建一个文本文档 (.txt)**。
2. 将其命名为 `classes.txt`。
3. 打开它，在**第一行**输入你的类别名称：
```text
mouse

```


*(注意：必须和你刚才标注时输入的单词一模一样，区分大小写！如果你标的是 `Mouse`，这里也要写 `Mouse`。根据你的报错，你应该写小写的 `mouse`)*
4. **保存**并关闭。

### 第二步：导入到 X-AnyLabeling

1. 回到 X-AnyLabeling 软件界面。
2. 看**顶部菜单栏**，点击 **File (文件)**。
3. 找到 **Import Class Names (导入类别名)** 或者类似的选项。
* *如果是中文版，可能是“导入标签列表”之类的。*
* *如果顶部菜单没找到，看看右侧侧边栏“Labels”列表上方有没有一个小小的“导入”图标或菜单。*


4. 选择你刚才创建的 `classes.txt` 文件。

### 第三步：再次导出

1. 导入成功后，你看右侧的标签列表里应该会出现 `mouse`。
2. 再次点击 **Export (导出)** -> **导出YOLO-Hbb标注**。
3. 这次应该就不会报错了，它会顺利生成文件夹。
