## <center>创建 YOLO 虚拟环境

### 第一步：配置国内加速源 （可选）

在 `cmd` 终端中，**依次执行**以下命令（逐行复制）：

```bash
conda config --set show_channel_urls yes
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/pytorch/
```
---

### 第二步：创建 YOLO 专属房间 (虚拟环境)

我们要给 YOLO 单独建一个房间，避免和别的项目冲突。

1. **创建环境：**
在 `cmd` 中输入（指定你自己的 Python 版本）：
```bash
conda create -n yolo python=3.13 -y
```


2. **进入环境：**
```bash
conda activate yolo
```

*注意：此时你的命令行最前面应该变成了 `(yolo)`，说明你已经进入房间了。*
（像下面这样）
```bash
(yolo) C:\Users\xxxx>
```
---

### 可能会出现的问题：
1. 运行`conda activate yolo`时输出如下：
```bash
CondaToSNonInteractiveError: Terms of Service have not been accepted for the following channels. Please accept or remove them before proceeding:
    - https://repo.anaconda.com/pkgs/main
    - https://repo.anaconda.com/pkgs/r
    - https://repo.anaconda.com/pkgs/msys2

To accept these channels' Terms of Service, run the following commands:
    conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/main
    conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/r
    conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/msys2

For information on safely removing channels from your conda configuration,
please see the official documentation:

    https://www.anaconda.com/docs/tools/working-with-conda/channels
```
* 这其实只是让你同意协议

- 法一：按照提示手动“同意”
依次在终端里运行下面这三行命令。

```bash
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/main
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/r
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/msys2
```

- 法二：清华源

既然我们要用清华大学的镜像源，其实根本不需要连接到官方的 `defaults` 源。我们可以直接**把它删掉**，这样既避开了这个报错，还能提高下载速度

**1. 找到配置文件**
* 找到配置文件 `.condarc`，通常在你的用户文件夹里。

* 路径一般是：`C:\Users\你的用户名\.condarc`


**2. 修改内容**
删除其中的内容，替换成下面这一段：

```yaml
channels:
  - https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main/
  - https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/
  - https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/pytorch/
  - https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/menpo/
  - https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/bioconda/
  - https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/msys2/
  - https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/conda-forge/
show_channel_urls: true
```

2. 运行`conda activate yolo`时输出如下：
`CondaError: Run 'conda init' before 'conda activate'` 是因为你的终端还没有被“授权”使用 conda 的切换功能

- 第一步：初始化终端
在当前的黑框框里，输入下面这行命令并回车：

```bash
conda init cmd.exe
```

- 第二步：重启终端
在新打开的窗口里，再次输入：

```bash
conda activate yolo
```

如果你的命令行最前面出现了 `(yolo)` 这样的字样，则说明虚拟环境配置成功
```bash
(yolo) C:\Users\xxxx>
```
